Appareil Photo Numérique

Description

Pilotage des appareils photo numériques via USB.

Liste des protocoles supportés :

  • Protocole standard PTP : ce protocole est supporté par la plupart des appareils numérique. Limitation : certains appareils implentent partiellement le protocole PTP . Par exemple le Canon 300D implemente les fonctions de rapatriement d'image, mais n'implémente pas les fonctions de déclenchement de pose par le cable USB. Dans ce cas, il faut utiliser le protocole propriétaire.
  • Protocole propriétaire CANON : Pour choisir le protocole , il faut utiliser le menu "communication" de l'appareil (NORMAL=protocole propriétaire, PTP=protocole PTP)
  • Protocole propriétaire NIKON : en chantier,en attendant utiliser le protocole PTP

Audela utilise la librairie OpenSource LIBGPHOTO2 v2.1.6 pour piloter les appareils photo numériques. Cette libriairie a été modifiée pour supporter les acquisitions en longue pose (voir intallation du patch 2.1.6d ci-dessous).

Voir la liste des appareils supportés : http://www.gphoto.org/

Remarque importante : L'écran de controle de l'appareil doit être éteint pendant que Audela dialogue avec l'appareil. On peut manipuler l'appareil entre deux acquisitions quand AudeLA n'est pas en train de dialoguer avec l'appareil.

Commande de création d'une caméra dslr

cam::create digicam {-name "xxxx" } {-gphoto2_win_dll_dir "xxxxxxx"}

Paramètre -gphoto2_win_dll_dir (défaut = "../bin")

Indique répertoire où se trouvent les fichiers de la librairies GPHOTO pour WINDOWS uniquement (fichiers libgphoto2.dll, libgphoto2_canon.dll, libgphoto2_port.dll,libgphoto2_port_usb.dll , libgphoto2_port_usb.dll , libgphoto2_ptp2.dll

Paramètre -name (défaut = "DSLR")

Indique le nom de la camera. "DSLR" est la seule valeur possible actuellement..

Exemples de programmation :

cam::create digicam
ou
cam::create digicam  -name DSLR

 

Commandes génériques ( cf. programmation des caméras )

    • acq
    • exptime
    • buf
    • stop
    • tel
    • close

Les commandes génériques non implémentées retournent des valeurs nulles.

 

Commandes spécifiques

cam1 autoload {0|1} (défaut = "1")

1=Active ou 0=désactive le chargemement automatique des images après la fin de chaque pose.

Si le chargement automatique est activé, l'image est chargée et supprimée de la mémoire de l'appareil photo. Si le chargement automatique est désactivé, l'image reste dans la mémoire de l'appreil. Il est possible de charger ultérieurement l'image avec la commande "cam1 loadlastimage"

Exemple de programmation:

cam1 autoload 1
cam1 acq  
wait cam1_status
confiVisu::autovisu 1  

cam1 autoload 0
cam1 acq  
wait cam1_status
....
cam1 loadlastimage
confiVisu::autovisu 1

cam1 drivemode {0|1|2} (défaut = "0")

Selection le mode d'acquisition de l'appareil :

  • 0 : normal (acquisition sans rafale et sans retardarteur).
  • 1 : rafale (acquisition de plusieurs images en rafale. Le nombre d'images acquises dépend des carctéristiques de l'appareil et de la durée de l'impulsion de déclenchement).
  • 2 : activation du retardateur interne de l'appareil.

    Exemple de programmation:

    cam1 drivemode 0
cam1 acq  
wait cam1_status
    confiVisu::autovisu 1

cam1 quality list

retourne la liste des qualités d'image disponibles sur l'appareil.

Exemple de programmation avec un Canon 300D :

cam1 quality list
   Large:Fine Large:Normal Middle:Fine Middle:Normal Small:Fine Small:Normal CRW

cam1 quality {xxxxx} (défaut = (*) )

Sélectionne la qualite d'image. Les valeurs possibles dépendent de la marque et des modèles. Il faut utiliser la commande "cam1 quality list" pour connaître les valeurs disponibles. L'ordre de changement de qualité est envoyé à l'appareil au lancement d'une acquisition avec la commande "cam1 acq"

(*) La valeur par defaut est premier élément retourné par la commande "cam1 quality list"

Exemple de programmation avec un Canon 300D :

cam1 quality list
   Large:Fine Large:Normal Middle:Fine Middle:Normal Small:Fine Small:Normal CRW    
cam1 quality Small:Normal
cam1 acq                  => l'appareil affiche le choix qualité=Small:Normal
cam1 quality CRW
cam1 acq                  => l'appareil affiche le choix qualité=CRW

cam1 loadlastimage

Charge la dernière image acquise présente dans la méoire de l'appareil. Cette commande est utile en particulier pour gagner du temps en chargeant une image pendant l'acquisition d'une image suivante en combinaison avec la commande "autoload 0".

Exemple de programmation : 

cam1 autoload 0 
cam1 exptime 300

cams1 acq
wait cam1_status
...
cam1 acq
cam1 loadlastimage
wait cam1_status
...
cam1 acq
cam1 loadlastimage
wait cam1_status
...
cam1 loadlastimage

cam1 systemservice {0|1} (défaut = 1 )

1=Active ou 0=désactive le service de détection automatique des appareils photo numériques géré par le système d'exploitation WINDOWS.

Il vaut mieux désactiver ce service pour éviter de perdre des images pendant leur rapatriement à cause d'un conflit d'accès entre Aud'ACE et d'autres logiciels qui utilisent ce service notamment l'explorateur de WINDOWS.

La commande "cam1 systemservice" sans parametre retourne la valeur courante

Remarques :

  • Equivalent à la commande WINDOWS : Panneau de configuration -> Outils d'administration -> Services -> Acquisition d'image Windows (WIA) -> Démarrer/Arrêter
  • Nécessite d'avoir des droits d'utilisateur étendus sous WINDOWS, sinon la commande reste sans effet.
  • N'est pas implémenté pour LINUX.

cam1 longuepose {0|1|2} (défaut = 0)

Sélectionne la liaison servant au déclenchement des acquisitions.

0 : Déclenchement des acquisitions via le câble USB de l'appareil. Pas de longue pose.
Déclenche le début de l'acquisition et attend que l'appareil signale lui-même la fin de l'acquisition.
Limitation : La durée d'acquisition est limitée sur certains appareils à 30 secondes

1 : Déclenchement des acquisitions via un autre lien que le cable USB de l'appareil (port parellele , quickremote, ...). Permet les longues poses. Le lien utilisé doit être indiqué avec la commande "cam1 longueposelinkno x"

2 : Déclenchement des acquisitions via un autre dispositif indépendant de Audela. Dans ce cas Audela ne déclenche pas la pose mais attend que l'appareil signale la fin d'une acquisition via le cable USB (attente de 120 secondes maximum)

La commande "cam1 longuepose" sans parametre retourne la valeur courante

Exemple d'utilisation : 

#--- declenchement par la liaison USB 
cam1 longuepose 0
cam1 acq
wait cam1_status
confiVisu::autovisu 1


#--- declenchement par le port parallele LPT1 ou /dev/parport1
cam1 longuepose 1
set linkno [link::create parallelport 1]
cam1 longueposelinkno $linkno
cam1 longueposelinkbit  3
cam1 longueposestartvalue 1
cam1 longueposestartvalue 0
cam1 acq
wait cam1_status
confiVisu::autovisu 1

#--- declenchement par quickremote
cam1 longuepose 1
set linkno [link::create quickremote 0]
cam1 longueposelinkno $linkno
cam1 longueposelinkbit  3
cam1 longueposestartvalue 1
cam1 longueposestartvalue 0
cam1 acq
wait cam1_status
confVisu::autovisu 1

#--- declenchement externe independant de Audela
cam1 longuepose 2
cam1 acq
wait cam1_status (attend qu'une pose soit déclenchée par une action independante de  AUDELA (par exemple appuyer sur le bouton l'appareil) 
confVisu::autovisu 1

cam1 longueposelinkno {xx} (défaut = 0)

Indique numero du lien qui sert au declenchement longue pose . Ce parametre est utilisé en combinaison avec "cam1 longuepose 1", il est ignoré dans les autres cas.

La commande "cam1 longueposelinkno" sans paramètre retourne la valeur courante.

Exemple d'utilisation : voir ci-dessus

cam1 longueposelinkbit {0...7} (défaut = 0)

Indique le numero du bit qui sert au déclenchement longue pose. Ce parametre est utilisé en combinaison avec "cam1 longuepose 1" et avec port parallele ou quickremote, il est ignoré dans les autres cas. .

La commande "cam1 longueposelinkbit" sans paramètre retourne la valeur courante.

Exemple d'utilisation : voir ci-dessus

cam1 longueposestartvalue {0|1} (défaut = 1)

Indique l'état du bit pour déclencher une longue pose. Ce parametre est utilisé en combinaison avec "cam1 longuepose 1", il est ignoré dans les autres cas.

cam1 longueposestartvalue 1 : déclenchement d'un acquisition quand le bit passe de 0 à 1

cam1 longueposestartvalue 0: déclenchement d'un acquisition quand le bit passe de 1 à 0

La commande "cam1 longueposestartvalue" sans paramètre retourne la valeur courante.

Exemple d'utilisation : voir ci-dessus

cam1 longueposestopvalue {0|1} (défaut = 1)

Indique l'état du bit pour arrêter une longue pose. Ce parametre est utilisé en combinaison avec "cam1 longuepose 1", il est ignoré dans les autres cas.

cam1 longueposestopvalue 1 : déclenchement d'un acquisition quand le bit passe de 0 à 1

cam1 longueposestopvalue 0: déclenchement d'un acquisition quand le bit passe de 1 à 0

La commande "cam1 longueposestopvalue" sans paramètre retourne la valeur courante.

Exemple d'utilisation : voir ci-dessus

 

Installation du patch LIBGPHOTO2 v2.1.6d

WINDOWS

Le patch est déjà pris en compte dans la version de LIBGPHOTO2 livrée avec Audela pour Windows.

Il faut installer "libusb-win32" pour utiliser LIBGPHOTO2 avec WINDOWS(voir le fichier readme.txt)

LINUX

Lors de l'installation de LIBGPHOTO2, il faut remplacer les fichiers *.c et *.h de LIBGPHOTO2 par ceux founis avec Audela dans le répertoire audela/src/external/libgphoto2/src

La copie des fichiers doit être effectuée après avoir lancé "./configure" et avant de lancer la compiltation "make" (voir fichier readme.txt)
