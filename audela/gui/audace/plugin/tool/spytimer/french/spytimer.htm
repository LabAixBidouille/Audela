SpyTimer

SpyTimer est un outil hybride comprenant :

  1. associé à EOS Utility (logiciel Canon), il permet de charger les images prises (RAW ou JPEG) et de les afficher dans Aud'ACE, au format initial ou au format fits avec les mots-clés choisis par l'utilisateur. Cette fonction est aussi utilisable même si Aud'ACE reconnaît l'APN.
  2. la définition des mots clés à inscrire dans l'en-tête FITS des images, y compris les paramètres optiques, bien que la caméra ne soit pas pas détectée par Aud'ACE.
  3. une aide à la focalisation.
  4. un timer-intervallomètre programmable, permettant de commander le déclenchement des longues poses des APN Canon, notamment ceux non reconnus par Aud'ACE, tels que 7D 60D etc. Il peut aussi commander les interfaces actionnées par un front montant ou descendant sur le port série.

Le matériel nécessaire est très limité (consulter l'aide disponible sous le menu Aud'ACE/Configuration/Caméra/APN) :

  1. la fonction de timer-intervallomètre nécessite un cordon spécial se connectant à la borne du récepteur de télécommande de l'APN. En l'absence de EOS Utility cette interface est limitée au déclenchement de l'APN, tous les paramètres de prise de vue étant réglés sur l'APN. Cette fonction est utilisable que l'APN soit reconnu ou non par l'OS.
  2. le réglage de l'APN, le déclenchement et le chargement via EOS Utility nécessitent uniquement le cordon de liaison USB entre le PC et l'APN (vendu avec). Cette fonction implique que l'OS et EOS Utility reconnaissent l'APN connecté, ce qui n'est pas toujours le cas (ex windows 7 ne reconnaît pas le 350D). EOS Utility donne également accès au mode LiveView, si l'APN en dispose, utile pour la mise au point.

Les deux outils peuvent utilisés isolément ou combinés. Dans les deux cas la configuration de la connexion de la caméra dans ll menu Configuration/Caméra... est sans objet.

1. Surveillance du répertoire images et utilisation de EOS Utility

Le principe est simple : EOS Utility déclenche la prise de vue et le chargement ; Aud'ACE fait le reste : surveillance du répertoire pour détecter le chargement, affichage de l'image, conversion de l'image au format FITS par défaut et incorporation des mots-clés utilisateur dans l'en-tête FITS. Voici comment procéder :

Ouvrir EOS Utility (appareil connecté ou non) ; cliquer sur le bouton Préférences d'EOS Utlity. Une fenêtre à onglets s'ouvre :

>
Onglet Dossier de destination C'est sans doute le point le plus délicat des réglages. Indiquer le chemin du répertoire des images. Attention : les images seront chargées dans un sous-dossier ; si ce sous-dossier n'existe pas il sera créé ; il est préférable de le créer manuellement car c'est ce sous-dossier qu'il faudra désigner comme répertoire images dans AudeLA. Le maintien de la date dans le nom du répertoire a pour effet de créer un nouveau dossier à 00h00m00s précises. Il est possible de personnaliser la création du dossier.
Onglet Nom du fichier Laisser en l'état ou personnaliser la genèse du nom de l'image, peu importe car la dernière image sera affichée quel que soit son nom.
Onglet Télécharger images Sélectionner l'option Images pas encore téléchargées.
Onglet Logiciel lié Sélectionner l'option Aucun. Ce ne serait pas une bonne idée de sélectionner audela.exe avec le bouton Inscrire ... pour ouvrir AudeLA avec EOS Utility car il ouvrirait des instances multiples.
Dans Aud'ACE : désigner le répertoire images dans le menu Configuration/Répertoires puis lancer le menu Caméra/SpyTimer.
Dans le panneau SpyTimerAud'ACE Pour activer la surveillance du répertoire, cocher l'option Surveiller le répertoire dans le sous-panneau de surveillance. Note : la procédure de surveillance dure une poignée de millisecondes (hors chargement de l'image). Le délai de surveillance est réglable dans le panneau de Configuration.
L'option Convertir permet de créer l'image au format fits par défaut dans le même répertoire image en préservant l'original. Les images FITS contiennent les mots-clés sélectionnés dans le panneau de Configuration.
Les extensions des images reconnues sont : CR2 CRW JPG cr2 crw jpg.

Conseils :

  1. Un test de prise de vue confirmera que les images sont chargées dans le bon dossier et que Aud'ACE affiche chaque nouvelle image.
  2. La cadence de prises de vues peut être très si élevée que AudeLA est incapable de suivre le rythme de conversion et d'affichage. Pour éviter ce problème AudeLA ne traite que la dernière image transférée.

2. Configuration

Le bouton Configuration ouvre un panneau comportant quatre sections de configuration :

1. Section de configuration des mots clés :
Un bouton donnant accès à la configuration des mots clés à intégrer dans l'en-tête FITS des images. Comme l'APN n'est pas reconnu par Aud'ACE, les valeurs des mots clés concernant le sytème optique seront nulles : des cases à cocher et des fenêtres de saisies des paramètres permettent de corriger cette lacune.
Le pouvoir séparateur sera calculé automatiquement. Le champ et l'échantillonnage sont calculés à chaque prise de vue en tenant compte des options en matière de qualité des images.
2. Section de configuration du déclenchement automatique de poses :
Une liste déroulante de sélection du Port. Les options disponibles sont identiques à celles proposées dans l'onglet Configuration/Caméra/APN.
Une liste déroulante de sélection du Bit de commande. Les options disponibles sont modifiées par le choix du port qu'il convient de régler en premier. La sélection d'un port LPT permet de choisir un bit entre 0 et 7 alors qu'un port COM permet de sélectionner une option parmi { DTR | RTS }.
Une liste déroulante de sélection de la commande d'activation Front. Les deux options possibles sont { 0 (front descendant) | 1 (front montant) }.
3. Une fenêtre de saisie du délai en secondes entre deux explorations du répertoire.
4. Une zone avec deux boutons, l'un activant l'aide à la focalisation (CF ci-après).

3. Focalisation

Le bouton Focalisation déclenche le suivi du répertoire, sans contraindre la conversion RAW->FITS, de manière à pouvoir utiliser des images JPEG dont le chargement est très rapide. Une fenêtre annexe presque identique à celle de l'outil Focalisation donne l'intensité et le contraste de l'image, la FWHM moyenne et le rapport FWHM x/ FHWM y d'une boîte de sélection (l'image entière par défaut).
La valeur maximale de l'axe des ordonnées de FWHM s'adapte à la valeur courante en masquant éventuellement des valeurs très supérieures.
Les index images vont de 1 à 20. Si ce nombre est dépassé, la fenêtre affiche les 19 dernières mesures. Un bouton Re-initialiser permet d'effacer la totalité des mesures.

4. Timer-Intervallomètre

L'outil comporte (du haut en bas), outre le bouton d'aide de l'outil, le bouton de Configuration et le sous-panneau de surveillance précités :

Une liste déroulante de sélection du Mode comporte deux options { Un | Série }.

  • Un (déclenchement) ne donne accès qu'au délai (retard) avant le déclenchement.
  • Série autorise l'entrée d'un intervalle entre deux déclenchements.

Une fenêtre d'affichage : Vide au repos, elle affiche les principales étapes au cours d'une session { Attente | Acquisition }.

Une fenêtre pour saisir le nombre de déclenchements. Le nombre est décrémenté en cours de session. A la fin de la session le nombre est remis à sa valeur initiale.

Une case à cocher Longue Pose : Lorsqu'elle est activée elle autorise la saisie d'une durée de déclenchement.

Une fenêtre de saisie de la Durée de l'impulsion. Celle-ci a de de l'importance :
- en mode Manuel + Bulb, car elle détermine la durée d'exposition,
- en mode Rafale, car elle détermine la durée de la rafale.
Dans tous les autres cas (Automatique, Manuel + Temps d'exposition) c'est le front montant ou descendant qui détermine le début du déclenchement mais c'est l'APN qui règle la durée. Le fait que la durée diffère du temps d'exposition, c'est un peu comme continuer à appuyer sur le bouton de déclenchement alors que la photo est déjà prise !

Une fenêtre de saisie du Délai avant le lancement du processus.

Une fenêtre pour saisir la Période de temps entre deux déclenchements pour l'option Série.
Cette entrée n'est active que si le nombre de poses est supérieur à 1.
Son calcul doit tenir compte de la durée, du retard éventuellement programmé sur l'APN et du temps de sauvegarde des images sur la carte mémoire.

Un bouton Go lance le processus. Toutes les commandes sont gelées durant une session : Aucun changement n'est possible.

Si l'option Auto est cochée les opérations débuteront à l'heure et à la minute spécifiée (heure civile). Inutile d'appuyer sur GO : le timer le fera.

Bon à savoir

  • L'interface ne peut pas savoir si vous avez mis en marche l'appareil, par exemple : allumé l'APN, mis une carte mémoire, etc. !
  • Chaque entrée est accompagnée d'un bulle d'aide.
  • Les données à saisir sont des nombres entiers et sont contrôlées. Une fenêtre rappelle les consignes en cas d'erreur.
  • Le lien créé avec le port n'existe que durant l'activation d'un déclenchement.
  • Chaque déclenchement est accompagné d'un message sur la Console indiquant le temps UTC, le numéro de série de la prise de vue, le numéro de l'image et éventuellement la durée d'exposition (Longue Pose). Ces informations figurent également dans le fichier C:\Documents and Settings\xxx\Application Data\AudeLA\log\ spytimer.log.
  • Les paramètres utilisateur sont sauvés dans le fichier C:\Documents and Settings\xxx\Application Data\AudeLA\ spytimer${N° visu}..
