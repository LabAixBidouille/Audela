ScopeDome

Les informations sur le dôme, les équipements et le driver sont sur le site de ScopeDome.
Jacek Pala, à contacter en anglais à l'adresse mail info@scopedome.com peut fournir les réponses aux questions.

Le driver, écrit en VBS, est ASCOM compliant et ne fonctionne que sous Windows.
Le driver dispose d'un mode Config/Card/Card Type = Simulator permettant de tester les fonctions.

L'interface ScopeDome de AudeLA est écrite en tcltk comme il se doit.
Le driver ScopeDome LS, version supérieure à 5.0, peut être lancé, lu, commandé et arrêté par cette interface.
La configuration du driver devra être réalisée en direct dans le driver.

Usages du driver

Lancement effectif du driver

Il faut indiquer le nom de l'exécutable et son chemin. Si le chemin est incorrect l'interface ne pourra pas être lancée. Ces deux paramètres sont conservés en mémoire.

Lors de l'appui sur le bouton Appliquer la version du driver est affichée, le driver est lancé et les commandes sont libérées. Le driver peut aussi être lancé automatiquement au démarrage dès lors que le nom et le chemin sont corrects.

Suivi du télescope

Le dôme peut suivre les évolutions du télescope piloté par une interface Aud'ACE : il suffit de cocher la case Suivre le télescope et les coordonnées actuelles sont transmises au driver ScopeDome LS qui doit être configuré correctement, onglet Telescope :

  • Scope Scan Interval : fréquence de refraîchissment (en secondes); rappel les coordonnées affichées dans le panneau télescope sont rafraichies environ toutes les secondes,
  • ScopeType : sélection de FileSystem,
  • chemin complet du fichier : C:\ScopeDome\ScopeDomeCurrentTelescopeStatus.txt (laisser inchangé).

Il faut également activer le bouton Scope Sync OFF qui bascule en Scope Sync ON.

Commandes en Tcl

Les commandes dupliquées dans l'interface sont regroupées en 4 lots :

Titre de la ligne nom de la variable du Sélecteur Valeur numérique
Valeur du senseur widget(property) widget(propertyResult) : valeur lue
Commande switch widget(switch) widget(switchValue) : valeur (entière) associée à la commande
Commande+double widget(cmd) widget(cmdValue) :valeur (double) associée à la commande
Commande widget(action)

Une commande console ou un script permet de passer des commandes au driver, par exemple :

set ::scopedome::widget(property) Dome_Scope_Is_Connected ; ::scopedome::cmd property ; # $widget(propertyValue) contient la valeur
set ::scopedome::widget(switchValue) On ; set ::scopedome::widget(switch) Scope_Sync ; ::scopedome::cmd switch ; #-- active le suivi du télescope par le dôme
set ::scopedome::widget(cmdValue) 245.5 ; set ::scopedome::widget(cmd) GoTo ; ::scopedome::cmd cmd ; #-- demande au dôme d'aller en position azimuth 245°30"
set ::scopedome::widget(action) Rel_Scope_On ; ::scopedome::cmd action ; #-- active le relai permettant d'alimenter le télescope

Arrêt du driver

L'appui sur le bouton Arrêter ou la fermeture d'Aud'ACE provoque la déconnection du driver et sa fermeture. Il est bon de vérifier si la carte est en mode StandBy ou normal.
