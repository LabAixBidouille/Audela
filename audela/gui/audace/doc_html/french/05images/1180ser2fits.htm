Conversion et extraction du format SER vers FITS

1. Analyse

AudeLA ne sait pas visualiser les images des films SER produits par certaines caméras (BASLER, iNova) utilisées en astronomie. Peu de logiciels savent décoder ce format.
Bien que limitées à 8 ou 12 bits, ces caméras sont intéressantes dans la mesure où elles disposent de réglages plus souples que les webcams, plus particulièrement en matière de rythme de prise de vue, de binning, d'AOI, etc.

2. Ce que fait cette fonction

Elle permet d'extraire une image, quelques images ou toutes les images en les convertissant au format FITS.
Les caméras BASLER d'Airylab produisent simultanément un fichier texte contenant bon nombre d'informations intéressantes (par exemple : nom du télescope, diamètre, taille des pixels, binning, temps d'exposition, heure de prise de vue, etc.).
Si ce fichier existe, la fonction en extrait les informations disponibles pour les transférer vers l'en-tête de l'image FITS.

Le format FITS des images monochromes dépend de l'enregistrement SER : Les enregistrements effectués en 8 bits sont convertis en images 8 bits (bitpix=8 ; FORMAT_BYTE), les autres en images 16 bits (bitpix=16 ; FORMAT_SHORT).

3. L'interface

Cette interface est structurée comme suit :
  • Un sélecteur de fichier SER contenu dans le répertoire images. L'éventuel fichier .ser.txt accompagnant le fichier .ser doit être dans le même répertoire.
  • Un sélecteur de la première et de la dernière image à convertir. Lors de la sélection d'un fichier, la fenêtre de la dernière image indique le nombre d'images contenues dans le fichier SER.
  • Un sélecteur de répertoire de destination : Un bouton annexe permet de le modifier (il doit exister).
  • La quatrième ligne permet de modifier le nom générique des images de sorties (par défaut celui du fichier SER). Les images produites ont un index identique au rang de l'image dans le fichier initial.
  • 3 ou 4 boutons pour les commandes habituelles.
