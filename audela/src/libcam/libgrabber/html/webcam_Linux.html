libgrabber - Linux V4L2 interface


The Linux implementation may interface with any V4L2 (Video for Linux 2) compliant kernel driver. The developper can find here some for details on the V4L2 API.

Disclaimer : this is an experimental piece of code that has not been extensively tested.

Main characteristics :

- Access modes : support of the Read and the Memory Mapped access modes. If the driver supports both modes, the Memory Mapped mode is chosen.
- Data format : for the Memory Mapped access mode, support of the RGB24, YUV422 and YUV420 data formats. For the Read acces mode, YUV422 data format.
- No support for the long pose mode of modified Philips webcams.
