AudeLA 

A software and a Tcl/Tk extension for astronomy

1. Introduction

AudeLA is a standalone software and an extension of the Tcl/Tk langage. 

As a Tcl/Tk extension, AudeLA provides a set of astronomical functions to drive modern instruments (telescopes and cameras). Celestial mechanics and image processing and analyzong are also included in the set of Tcl/Tk extension commands.

AudeLA is distributed with a graphical interface which is called Aud'ACE. By this way, AudeLA appears as a complete software directly usable by any astronomer.

Any Tcl/Tk script can be executed in a AudeLA process. It opens the possibility to personalize the software without any limit. All the devices of an observatory can be driven by AudeLA.

2. Using AudeLA as a standalone software

Download the last version of the binary installation. Linux and Windows versions are available.
This is a simpliest way to discover AudeLA and to use it for astronomical observations.
The planetarium software Cartes du Ciel must be also installed to provide a sky map to AudeLA.
The Aud'ACE graphical interface provides a window to display images and a terminal window to execute Tcl/Tk scripts.

AudeLA uses professional librairies to manipulate data (FITSIO, Sextractor) and can read stellar catalogs (USNO, UCAC, Tycho, NOMAD1 etc.)

AudeLA drives:
  • Telescopes : ASCOM, AudeCom, Celestron, DFM, EQMOD, Meade (LX200), Ouranos, Temma (Takahashi).
  • Cameras : Andor, DLSR, ASCOM, Atik, Audine, CookBook, FLI, Hi-SIS, Kitty, QSI, SBig, Starlight, Webcams.
  • Com interfaces : AudiNet, EthernAude,GPhoto2, Oscadine, Parallel  port, serial port, QuickRemote, Velleman
  • Focusers : AudeCOm, JMI, LX200, LhiresIII

3. Using AudeLA as a CGI in a web service

From the binary installation it is possible to deploy AudeLA as a CGI web service.
To deploy AudeLA as a CGI, execute the following command in the terminal window: 
source $audace(rep_install)/bin/cgi_install.tcl ; ::cgi_install::run
Fill the form and valid the Deploy button.
Read the help in the folder audela/gui/cgi

4. Write your own scripts to drive your observatory

The Tcl/Tk script langage provides the way to add new functions or to execute series a commands to drive the devices of your observatory all over the nights.
For example, to acquire 100 images of 10 seconds with binning 2x2, you can copy/paste the following lines in the audeLA terminal:
for {set k 1} {$k<=100} {incr k} {
   acq 10 2
   saveima image$k
}
As AudeLA provides celestial mechanics (ephemeris, coordinates tranforms, etc.) and image processing (dark, flat, registration, astrométric calibration, etc.) functions the script can detect the beginning and the end of the night and can process images during the night.

Read the tutorials to help you to write Tcl scripts.

5. Insert your own code in AudeLA

AudeLA allows you to add new commands using the Tcl langage but also using C code. For this last way you must download the source codes of AudeLA and install a compiler. To do that , read the file audela/doc/compil/audela_svn_windows_linux_en.odp. It explains how to download sources and the compilation step by step.
