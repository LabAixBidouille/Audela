Discrete Fourier Transform

The Discrete Fourier Transform (DFT) translates a spatial-domain image into two frequency (or Fourier) domain images. Mathematically speaking, this a complex transformation. So, depending on the output format, the frequency domain images will be different :

  • Cartesian format : the first output image will contain the real part of the DFT, while the second image will contain the imaginary part. The real part image will be an "even" image, as it shows some symetries with respect to the images axes. And the imaginary part image will be an "odd" image.
  • Polar format : the first output image will contain the modulus of the DFT, the second one the argument. The modulus image can be considered as the root square of the power spectrum of the original image. Hence it is a "even" image, while the other is an "odd" one.

With the order parameter, one can choose where frequencies are located in the resulting files :

  • Centered : The DFT low frequencies are gathered in the center neigthborhood of the image. The FITS header is updated with the DFT_ORD keyword which parameter is the string CENTERED.
  • Regular : The DFT high frequencies are gathered in the center neigthborhood of the image. The FITS header is updated with the DFT_ORD keyword which parameter is the string REGULAR.

The user may choose to process the displayed image, or to process a file stored on a hard disk.
When processing the previously displayed image, two resulting files will be created which names will depend on the selected format. If the format is set to polar, the files will be named as "modulus.fit" and "argument.fit". If the format is set to cartesian, the names will be "real.fit" and "imaginary.fit". If there were some files with the very same names, their content will be overridden.
If the user prefers to directly work on file, he will be able to select the names of the resulting files.
In both cases, when the processing is complete, the displayed image is the first output image, i.e., depending on the selected format, the real part or the modulus of the DFT operator.

The output images are encoded into unsigned 16-bits words. As the DFT result is signed, pixels values are shifted and normalised to fit the [0-32767] interval. The keywords DFT_OFFS and DFT_NORM contain respectively the shift and normalisation values.
